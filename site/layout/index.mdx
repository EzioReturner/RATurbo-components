协助进行页面级整体布局。

## 设计规则

### 尺寸

一级导航项偏左靠近 logo 放置，辅助菜单偏右放置。

- 顶部导航（大部分系统）：一级导航高度 64px，二级导航 48px。

- 顶部导航（展示类页面）：一级导航高度 80px，二级导航 56px。

- 顶部导航高度的范围计算公式为：48+8n。

- 侧边导航宽度的范围计算公式：200+8n。

### 交互

- 一级导航和末级的导航需要在可视化的层面被强调出来；

- 当前项应该在呈现上优先级最高；

- 当导航收起的时候，当前项的样式自动赋予给它的上一个层级；

- 左侧导航栏的收放交互同时支持手风琴和全展开的样式，根据业务的要求进行适当的选择。

## 组件概述

- `Layout`：布局容器，其下可嵌套 `Header` `Sider` `Content` `Footer` 或 `Layout` 本身，可以放在任何父容器中。

- `Header`：顶部布局，自带默认样式，其下可嵌套任何元素，只能放在 `Layout` 中。

- `Sider`：侧边栏，自带默认样式及基本功能，其下可嵌套任何元素，只能放在 `Layout` 中。

- `Content`：内容部分，自带默认样式，其下可嵌套任何元素，只能放在 `Layout` 中。

- `Footer`：底部布局，自带默认样式，其下可嵌套任何元素，只能放在 `Layout` 中。

## API

### 共同的API
| 属性 | 说明 | 类型 | 是否必须 | 默认值 |
| --- | ---- | --- | ------ | ------ |
| mode | 布局模式，垂直或水平 | vertical &#124; horizontal | 否 | vertical
| hideHeader | 隐藏header | boolean | 否 | false |
| fixHeader | 固定header | boolean | 否 | false |
| header | 替换header渲染内容，如不需要header，请使用 `hideHeader` | React.ReactNode | 否 | - |

### vertical模式API
| 属性 | 说明 | 类型 | 是否必须 | 默认值 |
| --- | ---- | --- | ------ | ------ |
| verticalType | 分列式布局或内联式布局 | split &#124; inline | 否 | split |
| collapsed | 折叠sider | boolean | 否 | false |
| hideSider | 隐藏sider | boolean | 否 | false |
| fixSider | 固定sider | boolean | 否 | false |
| sider | 替换sider渲染内容，如不需要sider，请使用 `hideSider` | React.ReactNode | 否 | - |

### horizontal模式API
| 属性 | 说明 | 类型 | 是否必须 | 默认值 |
| --- | ---- | --- | ------ | ------ |
| isContentFlowMode | 内容区域流式布局，为 `false` 时则定宽显示，由 `@layout-content-max-width` 控制  | boolean | 否 | true |
